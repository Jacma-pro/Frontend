# Framework

## Angular

- créé par Google en 2010
- framework front-end
- basé sur TypeScript

### Pourquoi Angular ?

- pratique pour des SPA (single page application)
- notion de modules et de composants
- libreries matérielles (Angular Material)

### Avantages d'Angular

- Architecture solide basée sur des composants
- Outils de développement puissants (CLI, DevTools)
- Intégration facile avec d'autres bibliothèques et frameworks
- Support à long terme de Google

### Les 3 grandes idées d'Angular
1. Composants : unités réutilisables de l'interface utilisateur
2. Templates : HTML avec des extensions Angular pour la liaison de données
3. Modules : regroupement de composants, services et autres fonctionnalités

### Différences avec React

Angular | React
--- | ---
Framework complet | Bibliothèque axée sur la vue
Basé sur TypeScript | Basé sur JavaScript
Utilise des templates en TypeScript | Utilise JSX

### trucs à savoir

- générer des composants avec Angular CLI : `ng generate component nom-du-composant`
- cycle de vie des composants : `ngOnInit`, `ngOnChanges`, `ngOnDestroy`
- gestion des formulaires : formulaires réactifs et basés sur les templates
- routage avec Angular Router pour la navigation entre les vues
